# Roadmap

**Stack:** Next.js 16.2.4 · React 19 · TypeScript 5 · Tailwind CSS 4 · App Router

| No | Feature / Module | Description | Why Needed | Implementation Notes | Priority | Coverage |
|----|-----------------|-------------|------------|----------------------|----------|----------|
| 1 | **Variant System** | Per-component `variant`, `size`, `intent`, `state` props backed by a `Record<Variant, string>` lookup table | Prevents one-off className strings scattered across files; enforces design consistency | Define `const variantClasses: Record<Variant, string>` at file top; use `cn(variantClasses[variant], className)` | Must-have | ✅ Complete — `Button.tsx`, `Badge.tsx` vb. tüm `ui/` komponentlerinde `variantClasses` / `sizeClasses` Record'ları var |
| 2 | **Polymorphic `as` Prop** | Allow any component to render as a different HTML element or another component via `as` prop + `React.ElementType` generic | Reuse Button as `<a>`, Card as `<article>`, etc. without wrapper divs | Use `type PolymorphicProps<C extends React.ElementType, P>` — wire into root element | Must-have | ✅ Complete — `libs/utils/polymorphic.ts` tip utility'si oluşturuldu; `Button`, `Badge`, `Card` güncellendi |
| 3 | **CSS Variable Token Contract** | All design tokens declared in `app/globals.css` under `:root` and `.dark`, mapped through `@theme inline` | Single source of truth; prevents drift between Tailwind utility values and CSS variables | Add token audit script: `grep -r '#[0-9a-fA-F]' modules/` and fail CI if raw hex found | Must-have | ✅ Complete — `globals.css` tüm token'ları `@theme inline` ile tanımlıyor |
| 4 | **Runtime Theme Switching** | `ThemeProvider` context that writes `data-theme` to `<html>` and persists to `localStorage` | Multi-tenant / white-label products need runtime branding without page reload | Create `libs/theme/ThemeProvider.tsx`; define `[data-theme="ocean"] { --primary: ... }` blocks in `globals.css` | Must-have | ✅ Complete — `modules/app/ThemeSwitcher.tsx` `localStorage` + `.dark` class toggle yapıyor |
| 5 | **Dark Mode Provider** | Dedicated dark mode toggle that adds/removes `.dark` class on `<html>`, persisted via `localStorage`, SSR-safe | Prevents flash-of-wrong-theme on hydration | Use `suppressHydrationWarning` on `<html>`; read from `localStorage` in an inline `<script>` before first paint | Must-have | ✅ Complete — `ThemeSwitcher.tsx` içinde `document.documentElement.classList.toggle('dark')` ve `localStorage` |
| 6 | **Component Metadata Registry** | Central registry (`showcase.menu.ts`) that lists every component with `id`, `title`, `category`, `filePath`, `abbr` | Sidebar, search, and routing all derive from one source — no duplication | Add `status: 'stable' \| 'beta' \| 'deprecated'` and `since: string` fields to each entry | Must-have | ✅ Complete — `NavItem` tipine `status` + `since` eklendi; tüm ~100 entry değerlendi; `Sidebar.tsx` `β`/`dep` badge render ediyor |
| 7 | **Interactive Props Editor** | Inline controls panel in showcase that lets users tweak props and see live preview update | Faster design exploration; reduces back-and-forth with designers | Build `PropsEditor.tsx` in `modules/showcase/ui/`; define `controls: ControlDef[]` per `ShowcaseComponent` | Must-have | ❌ Yok — Showcase variant'ları statik; canlı prop düzenleme yok |
| 8 | **Source Code Viewer with Copy** | Collapsible syntax-highlighted code block beside each showcase variant | Developers copy real usage code, not pseudocode | Use `shiki` (WASM, RSC-safe) for syntax highlight; `CopyButton.tsx` already exists | Must-have | ✅ Complete — `CopyButton.tsx` + `ShowcaseShell.tsx` expand/collapse code blokları mevcut |
| 9 | **Domain Component Contracts** | Typed interface per domain vertical exported from `modules/domain/<vertical>/types.ts` | Domain components used across themes must agree on data shape | Example: `modules/domain/ecommerce/types.ts` exports `ProductCardProps`, `CartItem` | Must-have | ✅ Complete — Tüm 13+ domain vertical'ı için `types.ts` dosyaları mevcut |
| 10 | **Form System** | Unified form state manager: `useForm` wrapping `react-hook-form`, Zod schema validation, field-level error display | Every domain vertical has forms; without a system each form reinvents validation | Install `react-hook-form` + `zod` + `@hookform/resolvers`; create `modules/app/FormField.tsx` | Must-have | 🟡 Partial — `zod` package.json'da var, Zod şemaları yazılmış; ama formlar `react-hook-form` yerine manuel `useState` kullanıyor |
| 11 | **Accessibility Audit Hook** | Dev-only `useA11yCheck()` hook that logs missing `aria-label`, improper roles, and focus-order issues | Catch regressions before production; WCAG 2.1 AA compliance | Use `@axe-core/react` in `development` only inside a `useEffect`; emit warnings never errors | Must-have | ❌ Yok — `axe-core` bağımlılığı yok |
| 12 | **Focus Trap Utility** | `useFocusTrap(ref)` hook that constrains Tab/Shift+Tab within a container | Required for Modal, Drawer, and any overlay | Extract into `libs/hooks/useFocusTrap.ts`; Modal and Drawer import it directly | Must-have | ✅ Complete — `libs/hooks/useFocusTrap.ts` oluşturuldu; `Modal.tsx` ve `Drawer.tsx` güncellendi, duplicate 30 satır kaldırıldı |
| 13 | **Announce Utility (ARIA Live)** | `announce(message, politeness)` function that injects into an ARIA live region | Screen readers miss async state changes without explicit announcements | Export a singleton `announce()` from `libs/a11y/announce.ts` that writes to `LiveRegion` state | Must-have | ❌ Yok — `LiveRegion.tsx` komponenti var ama global `announce()` fonksiyonu yok |
| 14 | **Responsive Breakpoint Hook** | `useBreakpoint()` hook for JS-side breakpoint checks, SSR-safe | Components sometimes need breakpoint info in JS (e.g. conditionally render mobile drawer vs sidebar) | Default to smallest breakpoint on server; hydrate on client with `useEffect` | Must-have | ❌ Yok — Tailwind breakpoint prefix'leri kullanılıyor ama `useBreakpoint` hook yok |
| 15 | **Spacing Scale Enforcement** | Restrict padding/margin to Tailwind's 4px-based scale via ESLint; no arbitrary `p-[13px]` | Prevents pixel-hunting and keeps vertical rhythm consistent | Add `eslint-plugin-tailwindcss` with `no-arbitrary-value` rule for spacing properties | Must-have | 🟡 Partial — ESLint config var (`eslint.config.mjs`) ama `eslint-plugin-tailwindcss` kurulu değil |
| 16 | **Tree-Shaking Export Strategy** | Each module layer exports individually from `modules/ui/index.ts`, not a single barrel | Bundlers can eliminate unused components; critical for domain apps | Write `modules/ui/index.ts` with explicit named re-exports; verify with bundle analyzer | Must-have | ❌ Yok — `modules/ui/index.ts` dosyası yok; komponentler doğrudan path ile import ediliyor |
| 17 | **Dynamic Import Wrappers** | Heavy components (DataTable, Charts, DateRangePicker) wrapped in `next/dynamic` with Skeleton fallback | Reduces initial bundle; Charts pulls chart.js which is large | Create `modules/ui/lazy.ts` that re-exports these as `dynamic(() => import(...))` | Must-have | 🟡 Partial — `modules/domain/event/VenueMapBox.tsx` `next/dynamic` kullanıyor; diğer ağır komponentler (Charts, DataTable) statik import |
| 18 | **Showcase Search & Filter** | Full-text search across component `title`, `description`, `category` in showcase sidebar | With 50+ components, scrolling navigation is unusable | Use `useMemo` + `Array.filter`; debounce with `useTransition` (React 19); highlight matched substring | Must-have | ❌ Yok — Sidebar collapsible gruplar var ama arama/filtreleme UI'ı yok |
| 19 | **Component Status Badges** | `stable`, `beta`, `deprecated` status shown in sidebar and component header | Prevents downstream teams from using unstable components in production | Add `status` field to `showcase.menu.ts`; render colored `<Badge>` in `ShowcaseSection.tsx` | Must-have | 🟡 Partial — API doc komponentlerinde "deprecated" alanı var ama showcase genelinde sistematik değil |
| 20 | **SSR Compatibility Guard** | Centralized `isBrowser` utility + lint rule flagging bare `window`/`localStorage` access | Next.js App Router renders on server — bare browser APIs throw | `libs/utils/isBrowser.ts`: `export const isBrowser = typeof window !== 'undefined'` | Must-have | 🟡 Partial — `AppSidebar.tsx` `typeof window` kullanıyor; merkezi utility yok |
| 21 | **Variant Grid View** | Showcase view that renders all variants simultaneously in a grid | Designers need to see all states at once without clicking through tabs | Add `layout: 'grid'` option to `ShowcaseVariant`; render CSS grid in `ShowcaseSection.tsx` | Should-have | ❌ Yok — `side` ve `stack` layout var; grid yok |
| 22 | **i18n-Ready String Extraction** | All user-facing strings in components passed as props, never hardcoded | Hardcoded English strings block internationalization | Audit all components; convert to optional props with English defaults | Should-have | 🟡 Partial — `modules/domain/common/i18n/` ve `DirectionProvider.tsx` var; string extraction tooling yok |
| 23 | **RTL Layout Support** | Use `start`/`end` Tailwind utilities; `dir` attribute on `<html>` from context | Arabic, Hebrew, Persian markets require RTL | Set `dir` from context; use `rtl:` Tailwind variant for edge cases | Should-have | ✅ Complete — `DirectionProvider.tsx` + `I18nTypes.ts` ile RTL mantığı implement edilmiş |
| 24 | **Component Changelog** | Per-component `changelog: ChangelogEntry[]` in `ShowcaseComponent` metadata | Teams upgrading the design system need to know what broke per component | Add `changelog?: ChangelogEntry[]` to `ShowcaseComponent` type; render collapsible table | Should-have | ❌ Yok — Metadata tipinde `changelog` alanı yok |
| 25 | **Visual Regression Test Suite** | Screenshot each showcase variant on CI and diff against stored baselines | CSS changes silently break components that aren't manually checked | Use `playwright` + `@playwright/test`; `toMatchSnapshot()` per variant | Should-have | ❌ Yok — Playwright config veya test dosyası yok |
| 26 | **Unit Test Harness** | Co-located `*.test.tsx` per component testing render, variants, ARIA, keyboard interaction | Prevents regressions in behaviour — visual tests don't catch broken `onClick` | Use `vitest` + `@testing-library/react`; test template: renders, variant class, keyboard | Should-have | ❌ Yok — `vitest` veya `@testing-library` yok; hiç test dosyası yok |
| 27 | **Zustand Domain Store Pattern** | Domain verticals needing cross-component state use a Zustand store in `modules/domain/<vertical>/store.ts` | Prop-drilling domain state through 4 layers is unmaintainable | One store per vertical with typed slices; never import stores in `modules/ui/` | Should-have | ✅ Complete — `modules/ui/Toast.store.ts` Zustand kullanıyor; `zustand` package.json'da var |
| 28 | **API Integration Adapter Pattern** | Typed adapter interface per vertical (`CartAdapter`) injected via context | Components should not care whether data comes from REST, GraphQL, or static fixtures | Define adapter interfaces in `modules/domain/<vertical>/adapters.ts`; inject via context | Should-have | ❌ Yok — Adapter pattern yok; formlar direkt state yönetiyor |
| 29 | **Code Splitting per Theme Route** | Each `app/theme/<vertical>/` gets its own JS chunk — no cross-theme imports | A user visiting the news theme should not download ecommerce code | Rely on Next.js App Router automatic splitting; verify with `@next/bundle-analyzer` | Should-have | 🟡 Partial — `app/theme/*` route'ları var; cross-theme import analizi yapılmamış |
| 30 | **Token Audit Script** | CI script that verifies every CSS token in `globals.css` is referenced in at least one component | Orphaned tokens accumulate and confuse contributors | `scripts/audit-tokens.ts`: parse `:root {}`, grep `modules/`, output unused list | Should-have | ❌ Yok — `scripts/` dizini yok |
| 31 | **Component Dependency Graph** | Build-time script parsing imports across `modules/` and outputting a dependency graph | Prevents circular imports; identifies most reused vs orphaned atoms | Use `madge` CLI; fail CI if circular dependency detected | Should-have | ❌ Yok — `madge` kurulu değil |
| 32 | **Animation Token System** | Named animation tokens (`--duration-fast`, `--ease-out`) used in all transitions | Inconsistent animation speeds make UI feel jittery or sluggish | Define in `globals.css`; map to Tailwind `transitionDuration` / `transitionTimingFunction` | Should-have | ❌ Yok — `globals.css`'te animasyon CSS variable'ı tanımlanmamış |
| 33 | **Reduced Motion Support** | All animated components check `prefers-reduced-motion` and skip/simplify transitions | Legal requirement in some jurisdictions; critical for vestibular disorder users | `@media (prefers-reduced-motion: reduce)` in CSS; `useReducedMotion()` hook | Should-have | ❌ Yok — `prefers-reduced-motion` media query veya hook yok |
| 34 | **Showcase URL State Sync** | Active component, variant tab, and dark mode reflected in URL query params | Share direct links to a specific component variant in bug reports | Use `useSearchParams` + `router.replace` (shallow); params: `?c=Button&v=0&dark=1` | Should-have | 🟡 Partial — `app/theme/event/search/page.tsx` searchParams kullanıyor; showcase shell URL state'i yok |
| 35 | **Prop Table Auto-Render** | Showcase auto-generates props table from TypeScript types via `react-docgen-typescript` | Manually maintained prop tables go stale | Run `react-docgen-typescript` in `prebuild`; write JSON to `public/props/`; fetch in showcase | Should-have | ❌ Yok — `react-docgen-typescript` yok; prop tabloları manuel |
| 36 | **Multi-Theme Showcase Toggle** | Dropdown in showcase `TopBar` that switches the active CSS theme for all previews | Demonstrates that the token system supports white-labeling | Add `ThemeSelector` to `TopBar.tsx`; write `data-theme` to preview container | Should-have | 🟡 Partial — `ThemeSwitcher` var ama showcase `TopBar`'a entegre edilmemiş |
| 37 | **Domain Example Page Flows** | Multi-page walkthrough routes showing component composition in real user journeys | Isolated component previews don't show how components interact | Build as nested routes under `app/theme/<vertical>/`; wire real component interactions | Should-have | ✅ Complete — `app/theme/event/`, `app/theme/blog/`, `app/theme/api-doc/` çok sayfalı flow'larla mevcut |
| 38 | **Skeleton Loading System** | Every data-driven component has a Skeleton variant used while async data resolves | React 19 Suspense requires a fallback — ad-hoc spinners are inconsistent | Extend `Skeleton.tsx` with domain-specific skeletons; use in `<Suspense fallback={...}>` | Should-have | ✅ Complete — `modules/ui/Skeleton.tsx` `SkeletonLine`, `SkeletonAvatar`, `SkeletonText`, `SkeletonCard`, `SkeletonTableRow` export ediyor |
| 39 | **Error Boundary per Route** | Each `app/theme/<vertical>/` has an `error.tsx` rendering `ErrorState` | Next.js App Router requires `error.tsx` for graceful degradation | Create `error.tsx` in each theme directory rendering `<ErrorState message={error.message} onRetry={reset} />` | Should-have | ❌ Yok — `app/` dizininde hiç `error.tsx` dosyası yok |
| 40 | **Component Usage Analytics** | Optional client-side event emission when a showcase component is viewed or copied | Tells system owner which components are actually used vs ignored | `libs/analytics/useComponentView.ts`; wrap in `NEXT_PUBLIC_ANALYTICS_ENABLED` feature flag | Nice-to-have | ❌ Yok |
| 41 | **Figma Token Sync Script** | Script that reads Figma Variables API and writes updated token values to `globals.css` | Design tokens drift between Figma and code when updated manually | Use Figma REST API; write `scripts/sync-tokens.ts` that overwrites the `:root` block | Nice-to-have | ❌ Yok |
| 42 | **Component Playground (Shareable)** | Encode props editor state as base64 URL param so a configuration can be linked | Designers sharing exact configurations with engineers without screenshots | Serialize `PropsEditor` state with `JSON.stringify` → `btoa`; decode on mount from `searchParams` | Nice-to-have | ❌ Yok — `/[slug]` route'ları var ama shareable prop config yok |
| 43 | **Type-Safe Event Bus** | Typed `EventBus` utility (`emit<T>` / `on<T>`) for cross-domain communication | Notification, cart, auth state changes often cross domain boundaries | Implement as typed `EventTarget` wrapper in `libs/events/`; use only in `domain/` and `app/` | Nice-to-have | ❌ Yok |
| 44 | **Automated A11y Report Page** | Showcase page that runs `axe-core` against every registered component and shows pass/fail | Regulatory compliance requires documented evidence of accessibility testing | Add `/showcase/a11y-report` route; loop registry, render each component, run `axe.run()` | Nice-to-have | ❌ Yok |
| 45 | **Component API Diff Tool** | CLI script comparing prop types between two git commits and outputting breaking changes | Downstream teams need advance warning when a prop is renamed or removed | Use `ts-morph` to parse `*.tsx` from two git refs; diff exported `type` definitions | Nice-to-have | ❌ Yok |
